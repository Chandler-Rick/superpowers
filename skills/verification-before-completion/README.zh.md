# 完成前验证

## 概述

未验证即声称完成，是不诚实而非高效。

**核心原则：** 先有证据，后有结论。

**违反规则即违反验证精神。**

## 铁律

```
未有新鲜验证证据，禁止声称已完成！
```

如未在本消息中运行验证命令，禁止声称通过。

## 核心流程

```
在声称任何状态或满意前：

1. 明确：什么命令能证明？
2. 执行：完整运行该命令
3. 阅读：完整输出，检查退出码，统计失败数
4. 验证：输出是否支持结论？
   - 否：据实说明并附证据
   - 是：带证据说明
5. 仅此时：可做出结论

跳过任何一步=撒谎而非验证
```

## 常见错误

| 声称 | 需有 | 不足之处 |
|------|------|----------|
| 测试通过 | 测试命令输出：0 失败 | 之前运行、“应该通过” |
| Linter 无误 | Linter 输出：0 错误 | 部分检查、推断 |
| 构建成功 | 构建命令：退出码0 | Linter 通过、日志正常 |
| Bug 修复 | 测试原始症状：通过 | 代码变更、假定修复 |
| 回归测试有效 | 红-绿循环验证 | 测试仅通过一次 |
| 代理完成 | VCS diff 有变更 | 代理报告“成功” |
| 满足需求 | 逐项核对清单 | 仅测试通过 |
