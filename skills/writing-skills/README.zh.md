# 编写技能

## 概述

**编写技能就是将测试驱动开发（TDD）应用于流程文档。**

**个人技能存放于代理专属目录（如 Claude Code 的 `~/.claude/skills`，Codex 的 `~/.agents/skills/`）**

你先写测试用例（子代理压力场景），观察其失败（基线行为），再写技能文档，观察其通过（代理遵循），最后重构（堵漏洞）。

**核心原则：** 未见代理因无技能而违规，不知技能是否有效。

**必备背景：** 使用本技能前，必须理解 superpowers:test-driven-development。该技能定义了 RED-GREEN-REFACTOR 基本循环。本技能将其应用于文档。

## 什么是技能？

**技能**是经过验证的技巧、模式或工具的参考指南，帮助未来的 Claude 实例找到并应用有效方法。

**技能包括：** 可复用技巧、模式、工具、参考指南

**技能不包括：** 仅讲述一次性解决方案的叙述

## TDD 与技能编写映射

| TDD 概念 | 技能创建 |
|----------|----------|
| **测试用例** | 子代理压力场景 |
| **生产代码** | 技能文档（SKILL.md） |
| **测试失败（RED）** | 无技能时代理违规（基线） |
| **测试通过（GREEN）** | 有技能时代理遵循 |
| **重构** | 堵新漏洞并保持合规 |
| **先写测试** | 写技能前先跑基线场景 |
| **观察失败** | 记录代理的具体借口 |
| **最小实现** | 针对性写技能堵漏洞 |
| **观察通过** | 验证代理遵循 |
| **重构循环** | 发现新借口→堵→再验证 |

整个技能创建过程遵循 RED-GREEN-REFACTOR。

## 何时创建技能

**适用：**
- 某技巧对你来说并非直观
- 你希望未来代理能复用此方法
